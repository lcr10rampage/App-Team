# Reusable Components

> Catalog of stable, reusable components. Before designing/building a new component, the Architect and Engineer check here **and** `app-build-set/app-system-guide.md` (which documents concrete components: navbar, stat cards, plan cards, modals, BookAutocomplete, full-screen animation shells) to reuse rather than reinvent. Preventing duplication is a named goal (see `memory/common-mistakes.md`).

## Entry format
```
### <Component name>
- Purpose:
- Where it lives (path):
- Approved use cases:
- Variants:
- Inputs / props:
- Dependencies:
- Accessibility notes:
- Known limitations:
- Projects using it:
```
