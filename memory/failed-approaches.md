# Failed Approaches

> Approaches that were tried and abandoned — so the team doesn't waste cycles re-attempting them. Complements the *Never Do Again* concept from `app-build-set/engineering-knowledge-prompt.md`. Manager-approved.

## Entry format
```
### <Approach attempted>
- Why it was chosen:
- What failed:
- Impact (time lost / damage):
- Replacement approach (what to do instead):
- When the failed approach might still be valid (edge cases):
- Confidence: ★★★★★
```
